# Handoff: 2台対戦 エアホッケー（キッズ向け）UI

## Overview
2台のiPadを同じWi-Fiでつなぎ、7歳と5歳のきょうだいが各自のiPadで対戦するエアホッケーゲームのUIです。世界観は「あたたかく・まるくて・ポップ」、絵本やおもちゃのような親しみやすさ。対象が未就学〜低学年のため **文字は最小限、色とアイコンで直感的に伝える** ことが最重要原則です。

本バンドルには次の4状態（3画面＋1演出）が含まれます。
- **① 対戦画面**（メイン）
- **得点シーン**（①の得点時のお祝い演出。①と同じ画面の状態違い）
- **② 接続画面**（2a えらぶ → 2b ホスト待機 → 2c ゲスト入力 → 2d つながった、の4ステップ）
- **③ 結果画面**

## About the Design Files
このバンドルの HTML ファイルは **HTMLで作成したデザインリファレンス**です。最終的な見た目と挙動を示すプロトタイプであり、そのまま本番コードとしてコピーするものではありません。

タスクは、これらのHTMLデザインを **ターゲットのコードベースの既存環境（React / Vue / SwiftUI / ネイティブ等）で、その確立されたパターンとライブラリを使って再現する**ことです。まだ環境が無い場合は、プロジェクトに最適なフレームワークを選定して実装してください。iPad（タブレット）の **横向き（ランドスケープ）** 前提で、2台がリアルタイム同期する設計です（同期の通信レイヤーは別途必要）。

## Fidelity
**High-fidelity (hifi)** です。最終的な配色・タイポグラフィ・余白・角丸・影・アニメーションを反映済み。既存ライブラリ・パターンを使ってピクセル単位で忠実に再現してください。キャラクターやアイコンは基本図形（円・角丸矩形・clip-path多角形）の組み合わせで描かれており、実装側でも同様にCSS/描画で再現するか、相当のアセットに差し替えてください（後述「Assets」参照）。

- プレイヤー青（スカイブルー）: `#4ea8de` / 明るい面: `#bfe0f5` / 頬: `#ff9d90`
- ミント（アクセント／ホスト・決定・もう1回ボタン）: `#3fc7a3`
- イエロー（アクセント／パック中心・王冠・星・得点ハイライト）: `#ffc233`

- 木枠（フレーム）グラデーション: `linear-gradient(#cda473, #bb874f)`
- 台のプレイ面（左右分割）: `linear-gradient(to right, #e7f3fb 0 50%, #ffeeeb 50% 100%)`
- 星バースト（薄イエロー）: `#ffe39a`

### Typography
- フォントファミリー: **"M PLUS Rounded 1c"**（Google Fonts、丸ゴシック）。weights: 500 / 700 / 800 / 900。
- フォールバック: `system-ui, sans-serif`
- スコア数字（大）: 900 / 54px（対戦時）・62px（得点ポップ時）/ line-height 1
- 結果スコア: 900 / 48px（勝者）・40px（敗者）
- ボタン主ラベル（もう1回 等）: 900 / 34px
- 接続ボタンラベル（へやを つくる 等）: 900 / 24px / line-height 1.15、中央揃え、改行で2行
- 合言葉タイル: 900 / 38px、ゲスト入力スロット: 900 / 32px、キーパッド: 900 / 28px
- 見出し（かち！等の演出文字）: 900 / 28〜32px、letter-spacing 2〜3px
- 補助文（まってるよ… / つぎは がんばろう！）: 800 / 15〜18px、opacity .6
- セクション見出し（キャンバス上のラベル。実装には不要）: 900 / 28px

### Spacing / Radius / Shadow（共通スタイル＝「ぷっくり」）
- **box-sizing: border-box を全体に適用**（これが無いと枠線・余白が加算されてはみ出す）
- 角丸: 画面カード `34〜38px`、ボタン `28〜30px`、タイル/キー `13〜20px`、台外枠 `28px`、プレイ面 `18px`
- 輪郭線: 主要素は `4px solid #1a2238`（小要素は3px）
- **ぷっくり影（最重要の質感）**: `box-shadow: 0 Npx 0 #1a2238` の「真下にずらした実線影」。
  - 大ボタン: `0 7〜8px 0 #1a2238`
  - マレット/スコアキャラ/タイル: `0 4〜5px 0 #1a2238`
  - 画面カード自体のドロップ影（キャンバス表示用、実装は任意）: `0 14px 34px rgba(26,34,56,.18)`
- タップ対象は大きく（子どもの指でも押せるよう、最小でも実寸44px以上。本デザインのボタン・キーは52〜74px相当）

### Animation（@keyframes）
- `spin`: 0→360deg 回転（ホスト待機スピナー、1.1s linear infinite）
- `hop`: translateY 0 →(45%)-16px→0（キャラがはねる。1〜1.3s ease-in-out infinite）
- `fall`: translateY(0)→640px かつ rotate 0→540deg（紙吹雪。各片で dur 2.4〜4.4s / delay 0〜3.2s をばらす）
- `pop` / `popbadge`: scale/rotate のポップ（得点数字・+1バッジ。0.6〜0.7s ease-in-out infinite alternate）
- `ping`: scale .4→1.8 / opacity .9→0（得点時にゴールから広がるリング。1s ease-out infinite）
- `float`: translateY 0→-8px→0（成功画面のキラキラ星。2.2〜2.8s ease-in-out infinite）
- `starspin`: 結果画面の背後の星バースト、ゆっくり回転（18s linear infinite）

## Screens / Views

### ① 対戦画面（メイン）
- **Purpose**: 対戦のメイン。各iPadが横向きで台全体を表示し、手前側を自分が操作する想定。
- **Layout**: 画面カード（横長 4:3、リファレンスでは 760×570px）。縦flex（column）で「上部スコアバー」＋「台（flex:1）」。
  - **上部スコアバー**: `space-between` の横並び。左＝青キャラ＋青の数字、中央＝一時停止ボタン、右＝赤の数字＋赤キャラ。padding 16px 26px 6px。
  - **台**: 木枠（`linear-gradient(#cda473,#bb874f)`、border 5px、角丸28px、影 `0 5px 0`）の内側に、border 3px・角丸18px・`overflow:hidden` のプレイ面。プレイ面は **左右分割**（左＝青 `#e7f3fb`／右＝赤 `#ffeeeb`）。
- **Components**:
  - **スコアキャラ（マスコット）**: 直径58px。色面の円＋濃紺輪郭＋ぷっくり影。目＝濃紺の縦長楕円×2、頬＝小判型×2（青キャラはコーラル頬、赤キャラはイエロー頬）、口＝下半円の笑顔（`border-bottom`のみの角丸）。青・赤で色違い。
  - **スコア数字**: 900 / 54px、青は `#4ea8de`、赤は `#ff6b5e`。
  - **一時停止ボタン**: 52px角・角丸16px・白地・濃紺輪郭・影 `0 4px 0`。中に濃紺の縦バー2本（8×21px）。
  - **ゴール（口が開いた形）**: 台の左右端中央。左ゴール＝高さ50%・幅50px・`#bfe0f5`・角丸 `0 60px 60px 0`（左辺なし）。右ゴール＝`#ffc9c2`・角丸 `60px 0 0 60px`（右辺なし）。内側にネット表現（`repeating-linear-gradient` の細い縞、opacity .5）。
  - **中央線**: 縦の破線（`border-left: 5px dashed #1a2238`、opacity .45）＋中央に破線の円（96px、`4px dashed`、opacity .4）。
  - **マレット**: 直径74px。色面円＋濃紺輪郭＋影 `0 5px 0`、中央に内側ディスク（38px、薄色面＋3px輪郭）。青マレットは左25%、赤マレットは右75%（top:50%）。
  - **パック**: 直径46px、濃紺の円＋影、中央にイエロー（18px）のドット。中央やや右（left:54%, top:46%）。
- **Content/copy**: なし（数字のみ。リファレンスは青3・赤5）。

### 得点シーン（①の状態違い・お祝い演出）
①と同レイアウトのまま、得点の瞬間を表現:
- 画面全体に **紙吹雪レイヤー**（`fall`、z-index上層、pointer-events:none）。コーラル/ミント/スカイ/イエローの小片（円 or 角丸矩形）を約18枚、位置・速度・回転をばらす。
- 得点した側のゴール口が **得点フラッシュ色 `#ffe39a`** ＋外側に黄色のグロー（`box-shadow: 0 0 0 4px rgba(255,194,51,.5)`）。
- ゴール付近に **pingリング**（黄色5px枠、`ping`）と、ゴール内に入った **パック**。
- **「+1」バッジ**: 黄色円58px・濃紺輪郭・影、`popbadge` で弾む。
- 得点側の **スコア数字が拡大ポップ**（例: 青4へ／62px・popbadge）し、相手側スコアは opacity .55 に沈める。
- 得点側の **マレットとキャラが `hop`** で跳ねる。
- ※リファレンスでは「青（左）が右ゴールに得点」した状態を表示。

### ② 接続画面（4ステップ）

**2a・えらぶ**
- 上部にロゴ（青マレット＋パック＋赤マレットの小アイコン横並び＋「エアホッケー」800/16px）。
- 下に **2つの大ボタンを左右並び**（各 flex:1、縦：アイコン＋ラベル中央）。

  - ゲスト＝イエロー `#ffc233`、アイコン＝白角丸72px（ドア＝右寄りの角丸枠＋差し込む左矢印）、ラベル「へやに / はいる」。
  - 両ボタンとも border 4px・角丸28px・影 `0 7px 0`。

**2b・ホスト待機**

**2c・ゲスト入力**

### ③ 結果画面
- **Purpose**: 1ゲーム終了後の勝敗発表。勝者を大きく祝い、敗者にもやさしく。
- **Layout**: 横長カード（760×570）を **横並び2カラム**。左（flex 1.15）＝勝者ショーケース、右（flex 1）＝スコア比較＋「もう1回」。画面全体に紙吹雪レイヤー（約30枚）。
- **左カラム（勝者）**:
  - 背後に **星バースト**（薄イエロー clip-path星 250px、`starspin` でゆっくり回転）。
  - 上に **王冠**（clip-path多角形。濃紺の下地＋イエローを内側に重ねて輪郭を表現、88×58px）。
  - 中央に **勝者キャラを拡大**（基準72pxを `scale(1.78)`、`hop`）。リファレンスは赤キャラ（目は大きめ、笑顔は塗りの口）。
  - 下に「**かち！**」900/32px・コーラル・letter-spacing 3px。
- **右カラム**:
  - **スコア比較**: 白カード2枚＋中央「VS」。勝者カードは `outline: 5px solid #ffc233` で強調（キャラ52px＋数字48px/コーラル）。敗者カードは通常（キャラ48px＋数字40px/スカイ）。各カード border 4px・角丸24px・影 `0 6px 0`。リファレンスは 赤7 - 青5。
  - **やさしいメッセージ**: 「あおチーム つぎは がんばろう！」800/15px・opacity .6・中央。
  - **「もう1回」ボタン**: 横幅いっぱい・ミント `#3fc7a3`・border 4px・角丸28px・影 `0 8px 0`。中に円弧＋三角の **リプレイアイコン**（濃紺）＋「もう1回」900/34px。

## Interactions & Behavior
- **接続フロー**: 2a で「つくる」→ 2b（合言葉を表示して待機、スピナー回転）／「はいる」→ 2c（合言葉入力、4桁そろうと決定可）。両者一致で 2d（成功演出「ピコン！」相当：チェックがhop＋星がfloat）→ ① へ遷移。
- **得点時**: ①→ 得点シーンの演出（フラッシュ／pingリング／+1ポップ／紙吹雪／キャラがhop／スコア拡大）。約1秒の演出後、パックをセンターに戻して続行。
- **試合終了**: 規定点（例: 7点）先取で ③ へ。勝者を `hop`＋王冠＋星バースト＋紙吹雪で祝う。
- **もう1回**: ③ の「もう1回」で ① に戻り再戦（スコアリセット）。
- **一時停止**: ① の一時停止ボタンでポーズ（メニュー等は本デザイン範囲外）。
- **アニメーション値**は上記 Design Tokens の @keyframes を参照（duration/easing 明記済み）。
- **タップ対象**: すべて大きく（44px以上）、色コントラストを明確に。

## State Management
- `screen`: `connect` | `playing` | `result`
- `connectStep`: `choose` | `hostWaiting` | `guestInput` | `connected`
- `role`: `host` | `guest`
- `roomCode`: 4桁（ホストが生成・表示／ゲストが入力。例 "4721"）
- `inputDigits`: ゲスト入力中の配列（0〜4桁）
- `scoreBlue` / `scoreRed`: 整数
- `winner`: `blue` | `red` | null
- `scoringSide` / `goalFlash`: 得点演出のトリガー（短時間true）
- `isPaused`: boolean
- **データ取得/通信**: 2台のリアルタイム同期（パック・マレット座標、得点、接続確立）が別途必要。WebRTC / WebSocket / ローカルネットワーク同期など、実装環境に応じて選定。本デザインは通信レイヤーを含みません。

## Assets
すべて **基本図形（円・角丸矩形）と CSS clip-path 多角形** で描画しており、外部画像アセットはありません。
- キャラ（マスコット）/ マレット / パック / ゴール / 一時停止・削除・決定・リプレイ・＋・ドア＋矢印アイコン: CSS のみ
- 王冠・星: `clip-path: polygon(...)`（READMEと元HTML内に座標あり）
- ネット表現: `repeating-linear-gradient`
- 紙吹雪: JSで生成した小要素（色・サイズ・速度・回転をばらす）
実装側でブランドのアイコン/イラストシステムがあれば差し替え可。フォントのみ外部依存（Google Fonts "M PLUS Rounded 1c"）。
